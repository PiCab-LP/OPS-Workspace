ShiftLog | Operapedia SystemShiftLog es una plataforma interna de gestión de incidentes y bitácora operativa diseñada para flujos de trabajo de alto rendimiento. El sistema permite el registro, seguimiento y resolución de incidentes en tiempo real, organizados por compañías y categorías personalizables.🚀 Características PrincipalesGestión de Incidentes (CRUD): Registro completo de eventos incluyendo descripción, acciones tomadas y estados (Pending, Important, Resolved).Persistencia Local: Utiliza un motor de datos basado en localStorage (vía DataStore), permitiendo que la información persista entre sesiones sin necesidad de una base de datos externa.Filtros Inteligentes: Buscador avanzado por palabras clave, filtrado por compañía, categoría y fecha específica.Sistema de Categorías: Gestión dinámica de categorías del sistema para adaptar la herramienta a diferentes necesidades operativas.Panel de Actividad (Logs): Historial en tiempo real de las últimas 50 acciones realizadas en el sistema (creación, edición, cambios de estado y borrados).Validaciones Operativas: Control de longitud de texto para asegurar que los reportes sean concisos (Límite de 50 palabras en descripción y 2000 en acciones).🛠️ Stack TecnológicoFrontend: HTML5 semántico y CSS3 moderno con variables de entorno para temas (Dark Mode por defecto).Lógica: JavaScript Vanilla (ES6+) con arquitectura orientada a objetos.Iconografía: Lucide Icons para una interfaz limpia y moderna.Almacenamiento: Browser Web Storage API (localStorage).📁 Estructura de ArchivosArchivoDescripciónindex.htmlEstructura principal, modales de sistema y contenedores de vistas.style.cssDiseño visual, animaciones de modales (overlay) y sistema de diseño.app.jsOrquestador de la UI, manejo de eventos del DOM y renderizado dinámico.data.jsCapa de datos (DataStore). El sistema asignará automáticamente la hora local y el turno.Resolver: Al marcar un incidente como "Resolved", el sistema solicitará obligatoriamente los detalles de la resolución final.Filtrar: Usa la barra lateral de filtros para segmentar la vista por fechas o compañías específicas.Actividad: Haz clic en el botón flotante "Activity" para desplegar el historial reciente de cambios.⚠️ Notas TécnicasGeneración de IDs: Los incidentes generan IDs únicos basados en timestamps codificados en base 36 (inc_lp9j8k...).Manejo de Tiempo: El sistema utiliza el timezoneOffset del navegador para asegurar que las horas registradas correspondan al horario local del operador, independientemente de la configuración UTC del sistema.
